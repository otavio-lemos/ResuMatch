# Design: Internacionalização (i18n) com next-intl

## Visão Geral

Implementar suporte a múltiplos idiomas (PT/EN) no ResuMatch usando a biblioteca `next-intl`, com detecção automática do navegador e opção de troca manual pelo usuário.

## Arquitetura

### Estrutura de Arquivos

```
├── i18n/
│   ├── request.ts      # Configuração de detecção de idioma
│   └── routing.ts      # Routing config (PT como default)
├── messages/
│   ├── pt.json         # Traduções português
│   └── en.json         # Traduções inglês
├── store/
│   └── useLanguageStore.ts  # Store Zustand para preferência manual
└── components/
    └── Navbar.tsx      # Flags funcionais
```

### Decisões de Design

1. **next-intl**: Biblioteca oficial para Next.js 15+ com App Router
2. **PT como padrão**: Idioma principal do app
3. **Detecção automática**: Accept-Language do navegador
4. **Preferência manual**: Sobrescreve detecção automática quando salva
5. **Migração gradual**: Componentes atualizados um por vez

## Instalação

```bash
npm install next-intl
```

## Configuração

### i18n/routing.ts
```typescript
import {defineRouting} from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['pt', 'en'],
  defaultLocale: 'pt'
});
```

### i18n/request.ts
```typescript
import {getRequestConfig} from 'next-intl/server';
import {routing} from './routing';

export default getRequestConfig(async ({requestLocale}) => {
  let locale = await requestLocale;
  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }
  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default
  };
});
```

### middleware.ts (opcional - para detecção automática)
```typescript
import createMiddleware from 'next-intl/middleware';
import {routing} from './i18n/routing';

export default createMiddleware(routing);
```

## Store de Preferência Manual

### store/useLanguageStore.ts
```typescript
import {create} from 'zustand';
import {persist} from 'zustand/middleware';

type Language = 'pt' | 'en';

interface LanguageStore {
  language: Language | null;
  hasUserPreference: boolean;
  setLanguage: (lang: Language) => void;
  clearPreference: () => void;
}

export const useLanguageStore = create<LanguageStore>()(
  persist(
    (set) => ({
      language: null,
      hasUserPreference: false,
      setLanguage: (language) => set({language, hasUserPreference: true}),
      clearPreference: () => set({language: null, hasUserPreference: false}),
    }),
    {name: 'language-storage'}
  )
);
```

## Padrão de Uso

### Em componentes cliente:
```tsx
import {useTranslations} from 'next-intl';

export default function MyComponent() {
  const t = useTranslations('nav');
  
  return <button>{t('dashboard')}</button>;
}
```

### Em componentes servidor:
```tsx
import {getTranslations} from 'next-intl/server';

export default async function ServerComponent() {
  const t = await getTranslations('sections');
  return <h1>{t('experience')}</h1>;
}
```

## Fluxo de Detecção de Idioma

1. Verifica se há preferência salva no localStorage
2. Se houver, usa a preferência do usuário
3. Se não, usa Accept-Language do navegador
4. Fallback para PT se nenhum idioma suportado

## Docker

Nenhuma mudança necessária no Docker - next-intl roda em runtime. O container reinicia automaticamente quando necessário.

## Cronograma

1. Configuração básica (next-intl + arquivos JSON)
2. Store de linguagem
3. Navbar funcional (flags clicáveis)
4. Editor (EditorPanel)
5. Dashboard (DashboardContent + ATSAnalysisView)
6. Demais páginas (modelos, import)
