# Code Review: ResumeMatch - AI Integration & Dashboard Enhancements

## 🛡️ Security Audit
*   [ ] **[Medium]** API Key in URL Query Parameter.
    *   *Location:* `app/actions/ai.ts:51`
    *   *Why:* The code uses `fetch(`${aiConfig.endpoint}?key=${aiConfig.apiKey}`, ...)` to call the Gemini API. While standard for this API, query parameters can be logged by browser history, server logs, or network proxies.
    *   *Recommendation:* Consider moving the API key to a `x-goog-api-key` header if supported by the endpoint version, or ensure this call only happens server-side (which it currently does via 'use server').

*   [ ] **[Low]** Limited Text Sanitization.
    *   *Location:* `app/actions/ai.ts:103`
    *   *Why:* Replaces only `{}`. 
    *   *Recommendation:* Ensure that the model is instructed to treat the input as data only, or use a more robust escaping method if injecting into complex prompts.

## 🐛 Logic & Correctness
*   [ ] **[Major]** Redirect Loop Risk in Navbar.
    *   *Location:* `components/Navbar.tsx:31`
    *   *Why:* The `useEffect` dependency array includes `pathname`. If a fetch failure or an empty list triggers a redirect, it might trigger the effect again.
    *   *Observation:* Currently safe as it only sets state, but worth monitoring for race conditions if more logic is added to that effect.

*   [ ] **[Minor]** Undo History state management.
    *   *Location:* `components/editor/EditorPanel.tsx:61`
    *   *Why:* `undoHistory` is a simple `Record<string, string>`. If a user triggers AI multiple times on the same field, only the *immediate* previous state is kept. 
    *   *Recommendation:* For a more robust "Undo", a stack per field would be better, but the current implementation satisfies the "last message" requirement perfectly.

## ♻️ Maintainability & Style
*   [ ] **[Nit]** Duplicated Code in `ai.ts`.
    *   *Location:* `app/actions/ai.ts:203-213`
    *   *Why:* `getResumeEditorSummaryContent`, `getResumeEditorRewriteContent`, and `getResumeEditorGrammarContent` all call `getAtsSummarySkill(language)`. 
    *   *Recommendation:* Since they currently share the same logic/skill block, they could be consolidated or at least point to a single internal helper to avoid repetition.

## 💡 Commendations
*   **Intelligent Sync:** The `syncAllModels` feature in the Zustand store is a great UX touch, allowing both simplicity for beginners and power for advanced users.
*   **Real-time Streaming:** The move away from "fake chat" to real SSE streaming in the Import Wizard significantly improves the perceived quality and transparency of the product.
*   **Safe Translation:** Creating a *copy* of the resume during translation instead of overwriting the original is the correct architectural choice to prevent data loss.

## 🏁 Final Verdict
**Approve** with nits. The system is much more robust now, and the separation of AI configurations follows the product's strategic needs.
